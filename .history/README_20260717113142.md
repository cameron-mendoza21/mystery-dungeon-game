This codebase is my work on a mystery dungeon based game. the main inspiration is the pokemon mystery dungeon games

I have no game development experience so this is all merely a hobby to touch base back with my coding skills i've learned over the years back in school.

im not quite sure if this project will be monetized. thats subject to if i ever get the impression this game is worth it.

The main concept of this game is a 2D Mystery Dungeon game. it features a Base where you can prepare you and your team for exploring dungeons, either to complete quests, or to make it to the end of the dungeon.

The design philosophy im going to keep it close to how the DS style mystery dungeon games were like. simple in gameplay loop, but complex enough to administer thought when dungeon crawling.

From this Github, i will be releasing just release version (.exe) where you can just run and it'll load the game up.

## Building Your First One-Room Dungeon (TileMap)

The project includes a starter tilesheet:

- `res://assets/tiles/starter_tilesheet.png`

Recommended first setup in Godot:

1. Create a new scene with `Node2D` as root (name it `DungeonTest`).
2. Add a `TileMapLayer` node (or `TileMap` if your Godot version shows that node).
3. In the inspector, create a new `TileSet` resource for that TileMap node.
4. Open the TileSet editor panel and add an Atlas source using `starter_tilesheet.png`.
5. Set tile size to `16x16`.
6. Create tiles from the atlas region.
7. Paint a simple room:
	- Floor in the middle.
	- Wall tiles around the edges.
	- Leave one wall gap for a doorway.
8. In the TileSet editor, add collision shapes to wall tiles only.
9. Place your player scene in the room and test movement/collision.

## Suggested Folder Pattern As You Grow

- `scenes/rooms/` for dungeon room scenes
- `scenes/actors/` for player/enemy scenes
- `scripts/player/` for player movement/state logic
- `assets/tiles/` for tilesheets and TileSet resources
